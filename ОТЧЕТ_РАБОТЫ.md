# Отчёт о проделанной работе
## Интеллектуальный дизайн и обратное восстановление составов оптических стекол

**Проект:** хакатон / исследование materials informatics  
**База данных:** SciGlass (GlassPy, `select_SciGK.csv.zip`), объединённый датасет `merged_data.parquet` (~425 293 записей)  
**Каталоги:** SCHOTT (122 стекла с полным набором nd, vd, ρ, Tg), файлы OHARA (в репозитории, не интегрированы в пайплайн)  
**Дата отчёта:** май 2026  

---

## 1. Цели проекта

### Основное направление
- Высокопреломляющие **бессвинцовые** стекла: **n_d > 1.80**, **PbO ≈ 0**
- Технологическая правдоподобность: стеклообразование, возможность промышленной варки

### Дополнительное направление
- Стекла с заданными оптическими параметрами (ν_d, дисперсия, прозрачность)
- Диагностика **обратной задачи**: насколько однозначно свойства → состав

### Жёсткие ограничения методологии
- **Не синтезировать** новые химические составы «из головы» вне данных: использовать только реальные записи SciGlass (matching) или кандидаты, явно помеченные как computational design с проверкой surrogate
- Всегда указывать **неопределённость** и число использованных соседей / distance_to_training

## 2. Данные и сопоставление свойств

### Источники
| Источник | Объём | Назначение |
|----------|-------|------------|
| SciGlass (zip) | ~422 949 стёкол | Поиск соседей, обучение моделей |
| `merged_data.parquet` | ~425 293 строк | ML, inverse design, GAN (колонки GlassPy + оксиды) |
| SCHOTT Excel | 122 стекла | Внешняя проверка recovery |

### Сопоставление имён свойств
| ТЗ / SCHOTT | SciGlass / merged_data |
|-------------|-------------------------|
| ND300 (n_d) | `RefractiveIndex` |
| NUD300 (ν_d) | `AbbeNum` |
| DENSITY | `Density293K` |
| TG | `Tg` |
| Tmelt | `Tmelt` (для аугментации Tg) |
| TLiq | `Tliquidus` |

### Оксиды
- **19 молярных** оксидов (SIO2, AL2O3, B2O3, …) — для проверки Σ ≈ 100 %
- **Групповые** (RO, R2O3, R2O5, RO2, Bi2O3, WO3, …) — в расширенных версиях v2/v3 и inverse design
- **La₂O₃, Nb₂O₅, Ta₂O₅** в SciGlass часто **не выделены** отдельно → прокси **R2O3 / R2O5**, **GEO2**

### Design space (high-n, lead-free)
- Условие: **n_d > 1.70**, **PbO ≤ 1 mol%**
- Размер: **~18 923** записей в `merged_data`
- С **n_d > 1.80** и PbO ≤ 1: **~12 462** реальных стёкол в SciGlass

## 3. Выполненные этапы и скрипты

### 3.1. Прямое ML: состав → показатель преломления
**Скрипт:** `train_refractive_index_models.py`  
**Наборы признаков:** элементы (80), молярные оксиды, весовые оксиды  
**Модели:** LinearRegression, Ridge, RandomForest, XGBoost, LightGBM, CatBoost  

**Результат (oxides, XGBoost, цель RefractiveIndex, фильтр 1.35 < n < 4):**
- R² ≈ **0.68**, RMSE ≈ **0.12**
- По **элементам** R² выше (~0.95) — из-за избыточности и корреляций, не как доказательство «лучшей физики»

**Выход:** `models/`, `models_oxides/`, `models_oxides_wt/`, `compare_models.py` → `figures/comparison/`

**Ограничение:** это **другая** целевая переменная и фильтр, не полный набор ND300/NUD300/DENSITY/TG из ТЗ.

---

### 3.2. Version 1 — SCHOTT → SciGlass matching (baseline)
**Скрипт:** `match_schott_sciglass.py`  
**Метод:** k=20 соседей, метрики: Euclidean (z-score), cosine, Mahalanobis; состав = **среднее** соседей  

| Параметр | Значение |
|----------|----------|
| SciGlass с 4 свойствами (strict) | 4 474 |
| SCHOTT обработано | 122 |
| Признаки | ND300, NUD300, DENSITY, TG |

**Выход:** `output/schott_match/`

**Недостатки (выявлены в v2/v3):**
- Усреднение составов → **физически недопустимые** смеси
- `physically_plausible` на смеси ≈ **0%**
- Узкий пул strict

---

### 3.3. Version 2 — adaptive pool, взвешенные составы, uncertainty
**Скрипт:** `match_schott_sciglass_v2.py`  
**Улучшения:**
- STRICT vs **ADAPTIVE** (≥3 из 4 свойств) → пул **20 195** (×4.5)
- Расстояние с штрафом √(4/n_used)
- Состав: exp-weight, IDW, cluster (KMeans/HDBSCAN) вместо простого mean
- `composition_completeness`, uncertainty, MODE_A vs MODE_B

**Ключевой результат:**
- **composition_shift (median) = 7.41** между MODE_A (только n_d, ν_d) и MODE_B (+ρ, T_g)
- Добавление двух свойств **радикально меняет** «ответ» по составу

**Выход:** `output_v2/`

---

### 3.4. Version 3 — обусловленность обратной задачи (главный научный результат recovery)
**Скрипт:** `match_schott_sciglass_v3.py`  

| Метрика | Значение | Интерпретация |
|---------|----------|----------------|
| **Jaccard** top-20 (MODE_A ∩ MODE_B) | **median 0.026** | Множества соседей почти не пересекаются |
| Ill-posed (J < 0.3) | **99.2%** | Обратная задача неустойчива |
| best_neighbor plausible | **~11%** | Отдельные соседи физичны |
| plausible_ratio top-20 | **~13%** | Смешивание ломает физику |
| Вклад nd в d² | **~23%** | Нет доминирования одного признака |

**Состав:** только **лучший сосед** (без усреднения); локальные PLS/XGB на top-20.

**Выход:** `output_v3/`

**Вывод для статьи:** recovery по 4 макросвойствам — не «плохая модель», а **структурно недоопределённая** задача; степень неоднозначности **измерима** (Jaccard).

---

### 3.5. Inverse design — surrogate + NSGA-II
**Скрипт:** `inverse_glass_design.py`  

**Прямые модели (composition → property), XGBoost/CatBoost:**

| Свойство | R² (test) | RMSE | Замечание |
|----------|-----------|------|-----------|
| ND300 (n_d) | 0.68 | 0.12 | Приемлемо |
| NUD300 (ν_d) | 0.90 | 5.4 | Хорошо |
| DENSITY | **0.20** | 1.12 | **Слабо** |
| TG | 0.81 | 83 °C | Средне; см. аугментацию Tg |

**Аугментация Tg:** Tg ≈ (2/3)·T_m в **Kelvin**; +10 325 imputed точек; вес imputed = 0.4  
**Проверка на SciGlass:** median(Tg_K/Tm_K) ≈ **0.67**; средняя ошибка правила vs измеренный Tg ≈ **50 °C**

**NSGA-II (прогон 2h: pop=400, gen=520):**
- Pareto: 400 точек; **feasible: 88**
- Top-20: PbO=0, **n_pred 1.95–2.19**, median distance ≈ **1.7**
- Инициализация из **реальных** составов SciGlass

**Выход:** `output/inverse_design_2h/`, графики `design_evaluation_summary.png`, `pareto_nd_vd.png`

**Это design, не recovery:** успех = кандидаты в manifold + ТЗ по surrogate, не совпадение с одним эталонным стеклом.

---

### 3.6. cWGAN-GP — генеративный design
**Скрипт:** `gan_glass_design.py`  
**Метод:** conditional Wasserstein GAN + gradient penalty; условие (n_d, ν_d); PbO=0 в генераторе  

**Обучение:** 600 эпох, ~13 669 составов с n и ν_d; CPU  

| Метрика | GAN | NSGA-II (2h) |
|---------|-----|----------------|
| Сгенерировано | 4 980 | 400 |
| Feasible | **2 010 (40%)** | 88 (22%) |
| Max n_pred | 2.17 | 2.38 |
| Median distance | 1.38 | ~1.3 (feasible) |

**Выход:** `output/gan_design/`, `nsga_vs_gan.png`, `gan_nd_vd_scatter.png`

**Наблюдение:** GAN даёт **больше** feasible точек **внутри** плотного облака SciGlass; NSGA-II — **выше n** и шире по novelty.

## 4. Сводная интерпретация

### Что удалось доказать
1. **Обратная задача (properties → composition) для каталогных стёкол — ill-posed** (Jaccard ≈ 0, shift ≈ 7.4).
2. **Прямая задача (composition → n_d, ν_d)** обучается с R² ~0.7–0.9 на больших данных.
3. **Computational design** (NSGA-II, GAN) даёт **списки кандидатов** под ТЗ (Pb-free, n > 1.8) в окрестности известных high-n стёкол SciGlass.
4. **Усреднение соседей** — методологическая ошибка для составов; **лучший сосед** или **Pareto/GAN** — корректнее по смыслу.
5. Для **n > 1.8 без Pb** пространство **не пустое** (~12k реальных точек), но **сжимается** к TiO₂/ZrO₂/Bi₂O₃/WO₃ и групповым R₂O₃/R₂O₅.

### Что не удалось / не делалось
- **Лабораторная** проверка плавки и измерения n_d, ν_d
- **Интеграция OHARA** в единый пайплайн сравнения (файлы есть, скрипт не написан)
- **CALPHAD / QComp** / стеклообразование из SciGlass как жёсткий фильтр
- **Восстановление** точного состава SCHOTT-стекла с химической точностью
- **Отдельные оксиды** La₂O₃, Nb₂O₅, Ta₂O₅ на уровне данных
- **Спектральная** прозрачность, TEC*, полный набор V(λ)
- **Валидация GAN** на mode collapse (уникальность составов) — не формализована
- **Экспертная** рецензия top-20 кандидатов материаловедом

## 5. Ограничения данных

1. **Разрежённость свойств:** из 422k записей полный набор (n, ν_d, ρ, T_g) — **~1%** (strict pool).
2. **Групповые оксиды** (R2O3, R2O5, RmOn) завышают `composition_completeness` и размывают химию.
3. **merged_data** vs zip SciGlass: разные схемы имён колонок; риск несогласованности при смешивании.
4. **Surrogate ≠ физика:** предсказанные n_d > 2.3 возможны при экстраполяции; DENSITY почти не предсказывается (R²=0.2).
5. **SCHOTT 122 стекла** — малый тест для статистики recovery; выводы о Jaccard — сильные, но на одном каталоге.
6. **Tmelt → Tg:** правило 2/3 даёт ~50 °C MAE; imputed точки не заменяют измерения.

## 6. Ограничения методов

### Matching (v1–v3)
- Метрика в **z-пространстве** 4 скаляров не отражает химическое расстояние.
- **k=20** — эвристика; нет теоремы об оптимальности k.
- **Mahalanobis / cosine** в v1 не изменили качественный вывод v3.
- Локальные модели (PLS/XGB) на 20 точках — **переобучение**, CV MSE высокий.

### NSGA-II
- Оптимизация **surrogate**, не реальных свойств.
- Feasible зависит от порога `distance_to_training` (p95 LOO ≈ 1.41).
- Серая «хвостовая» зона Pareto (distance до 10) — экстремумы модели, не стёкла.
- Не учтены: TEC, T_liq в оптимизации (модель TLiq опциональна, слабо валидирована).

### cWGAN-GP
- Обучение **только на** точках с n и ν_d (**13 669**, не 18 923 design space).
- **G loss растёт** (180+ к концу) — типично для WGAN; качество судить по **фильтрованным** кандидатам.
- Условность по (n_d, ν_d) **не гарантирует** достижение цели после surrogate (n_pred cap ~2.17 vs NSGA 2.38).
- Долгое обучение на CPU (~2 ч на 600 эпох).
- Нет сравнения с **VAE** или **diffusion** (не реализовано).

### Общее
- Все кандидаты — **гипотезы** до эксперимента.
- **distance_to_training** — евклидово расстояние в пространстве оксидов после StandardScaler, не Mahalanobis по свойствам.
- **Нет uncertainty quantification** у XGBoost (нет интервалов, только эвристики v2).

## 7. Недостатки и риски для хакатона / публикации

| Риск | Серьёзность | Комментарий |
|------|-------------|-------------|
| Surrogate extrapolation | Высокая | Высокий n_pred может быть артефактом |
| DENSITY R²=0.2 | Высокая | Pareto по ρ ненадёжен |
| 0% plausible на смеси | Средняя | Исправлено в v3, но важно для v1/v2 отчётов |
| GAN vs NSGA не сверены с OHARA | Средняя | Нет внешней валидации |
| Групповые оксиды в top-кандидатах | Средняя | R2O3+R2O5 > 50% — трудно варить без расшифровки |
| Один random seed | Низкая | NSGA/GAN не повторялись |
| joblib/loky warnings на Windows | Низкая | Прерывали scoring GAN; исправлено `--score-only` |

## 9. Рекомендации по использованию результатов

### Для отчёта / статьи (recovery)
- Центральный тезис: **измеримая неоднозначность** (Jaccard, composition_shift).
- Не заявлять «восстановили состав SCHOTT N-XXX».
- Показать: `output_v3/figures/jaccard_histogram.png`, `distance_attribution.png`.

### Для ТЗ (design Pb-free, n > 1.8)
- Таблица top-20 из `inverse_design_2h` **и** `gan_design` с колонками: composition, n_pred, νd_pred, Tg_pred, distance, novelty class.
- Для каждого кандидата: **ближайший ID в SciGlass** (KOD/GLASNO) и класс: duplicate / variant / novel.
- Сверка 3–5 позиций с **OHARA** вручную.

### Следующие шаги (если продолжать)
1. Скрипт `evaluate_design_candidates.py` (novelty, nearest SciGlass, OHARA).
2. Переобучить DENSITY на подмножестве high-n.
3. Жёсткий фильтр: только кандидаты с **явными** TiO₂/ZrO₂/Bi₂O₃/WO₃ в пределах p95, без опоры только на R2O3/R2O5.
4. Экспертная оценка 5 составов.

## 10. Команды запуска (кратко)

```bash
# Matching v3 (диагностика recovery)
python match_schott_sciglass_v3.py

# Inverse design (NSGA-II)
python inverse_glass_design.py --output output/inverse_design_2h --pop-size 400 --generations 520

# GAN (после обучения — только scoring)
python gan_glass_design.py --score-only --forward-models output/inverse_design_2h/forward_models

# Forward models с Tg augmentation
python -c "from pathlib import Path; from inverse_glass_design import *; df=load_sciglass_df(Path('merged_data.parquet')); train_forward_models(df, oxide_columns(df), Path('output/inverse_design_tg_aug'))"

# ML RefractiveIndex
python train_refractive_index_models.py --features oxides
python compare_models.py
```

## 11. Итог одной фразой

**Прямой путь «свойства → один истинный состав» для оптических стекол по каталогным макросвойствам не работает (ill-posed); работает постановка «свойства → семейство + набор реальных кандидатов + метрика неопределённости» и отдельно — «computational design кандидатов внутри эмпирического manifold SciGlass» с последующей экспериментальной проверкой.**
